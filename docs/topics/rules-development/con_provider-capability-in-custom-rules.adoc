= Provider capabilities in custom rules

[role="_abstract"]
In a rule, the `when` block is where the conditions for matching the rule are specified. Each provider offers a series of capabilities to do matching.

The search query in the rule condition can contain patterns, code locations, specific dependencies to be found, and so on, to evaluate the source code and dependencies. The provider sends the  Language Server Protocol (LSP) server a request to check the search query against the application being analyzed. When the LSP server returns a match for the search in the source code, the analyzer triggers a violation.

The syntax of the when block is as follows: it contains a single condition, but that condition might have multiple conditions nested beneath it.

----
when:
  <condition>
    <nested-condition>
----
