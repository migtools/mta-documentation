= Tag actions

[role="_abstract"]
A tag action instructs the analyzer to generate one or more tags for the application when it finds a match. Each string in the `tag` field can be a comma-separated list of tags. Optionally, you can assign categories to tags.

[source,yaml, subs="+quotes"]
----
tag:
  - "_<tag1>_,_<tag2>_,_<tag3>_"
  - "Category=_<tag4>_,_<tag5>_"
----

_Example_

[source,yaml, subs="+quotes"]
----
- ruleID: test-rule
  when:
    _<CONDITION>_
  tag:
  - Language=Golang
  - Env=production
  - Source Code
----

A tag can be a string or a `key=val` pair, where {ProductShortName} treats the key as a tag category. You can consider any rule that has a tag action as a “tagging rule” in this document.

[NOTE]
====
Issues are not created for rules that contain only tag actions.
====
